一、什么是虚拟列表？

虚拟列表是一种只渲染可视区域内元素的性能优化技术。当列表数据量很大时（如上万条），如果全部渲染到 DOM，会导致页面卡顿甚至崩溃。虚拟列表的核心思想是：

无论数据有多少，DOM 中只保留用户能看到的那几十条元素。

不定高虚拟列表（复杂版）

每个列表项高度不固定，可能因为内容不同而高度各异。这带来几个核心难题：

  1. 无法预知总高度：因为不知道每个项的实际高度
  2. 无法直接定位：不能通过简单除法计算索引
  3. 需要动态测量：渲染后才能知道真实高度
  4. 位置需要更新：测量后要重新计算所有后续项的位置

九、总结

核心要点

  1. 预估 + 修正策略：先用默认高度预估，渲染后测量真实高度并修正
  2. 二分查找定位：解决不定高无法直接计算索引的问题
  3. 位置缓存：用 Map 存储每项的位置信息，避免重复计算
  4. 批量更新：使用 RAF 合并高度更新，减少重计算次数
  5. 缓冲区：上下多渲染几项，避免滚动时出现白屏

适用场景

  • 聊天消息列表（消息长度不一）
  • 商品列表（商品信息高度不同）
  • 新闻列表（标题和摘要长度不同）
  • 评论区（评论内容长度不同）

性能优化建议

  1. 合理的默认高度：尽量接近真实平均高度，减少修正幅度
  2. 适当的缓冲区：bufferCount 太小会白屏，太大浪费 DOM
  3. 避免频繁更新：使用 RAF 批量处理高度变化
  4. 及时清理：组件销毁时清理 ResizeObserver 和 Map

扩展思考

如果需要支持动态增删列表项，还需要考虑：

  • 插入/删除时更新索引
  • 重新计算受影响项的位置
  • 维护 itemsOrderKeys 数组
